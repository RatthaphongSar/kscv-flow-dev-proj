3. API ENDPOINTS REQUIREMENTS
================================================================================

3.1 Backend API Routes - Authentication & Registration

   POST /api/registration/auth/send-otp
   - Purpose: Send OTP to email for verification
   - Body: { email: string }
   - Response: { success: boolean, message: string, expiresIn: number }
   - Status: 200 | 400 (invalid email) | 429 (rate limit)
   - HTTPS: ✓ Required

   POST /api/registration/auth/verify-otp
   - Purpose: Verify OTP and generate temp session token
   - Body: { email: string, otp: string }
   - Response: { success: boolean, tempToken: string, expiresIn: number }
   - Status: 200 | 400 (invalid OTP) | 401 (expired)
   - HTTPS: ✓ Required

   GET /api/registration/check-email/:email
   - Purpose: Check if email already registered
   - Response: { exists: boolean, type: string (STUDENT|TEACHER|NONE) }
   - Status: 200
   - HTTPS: ✓ Required

3.2 Student Registration Endpoints

   POST /api/registration/student/validate-id
   - Purpose: Validate Thai Student ID against existing students
   - Body: { studentId: string }
   - Response: { valid: boolean, name: string (if found), message: string }
   - Status: 200
   - HTTPS: ✓ Required

   POST /api/registration/student/submit
   - Purpose: Submit student registration form
   - Auth: Temp token from OTP verification OR empty
   - Body: FormData {
       email: string,
       phone: string,
       firstName: string,
       lastName: string,
       studentId: string,
       dateOfBirth: string (ISO),
       address: string,
       files: FileList (identification documents)
     }
   - Response: { submissionId: string, status: string }
   - Status: 201 | 400 | 401 | 422
   - HTTPS: ✓ Required

   GET /api/registration/student/status/:submissionId
   - Purpose: Check registration status
   - Response: { status: string, message: string, rejectionReason?: string }
   - Status: 200 | 404
   - HTTPS: ✓ Required

3.3 Teacher Registration Endpoints

   POST /api/registration/teacher/validate-id
   - Purpose: Validate Thai National ID or Employee ID
   - Body: { idNumber: string }
   - Response: { valid: boolean, suggestions?: string[], message: string }
   - Status: 200
   - HTTPS: ✓ Required

   POST /api/registration/teacher/submit
   - Purpose: Submit teacher registration form (multi-step)
   - Auth: Temp token from OTP verification
   - Body: FormData {
       email: string,
       phone: string,
       firstName: string,
       lastName: string,
       idNumber: string,
       dateOfBirth: string,
       address: string,
       educationLevel: string,
       specialization: string,
       experience: number,
       certifications: FileList,
       documents: FileList
     }
   - Response: { submissionId: string, status: string }
   - Status: 201 | 400 | 401 | 422
   - HTTPS: ✓ Required

   GET /api/registration/teacher/status/:submissionId
   - Purpose: Check teacher registration status
   - Response: { status: string, message: string, rejectionReason?: string }
   - Status: 200 | 404
   - HTTPS: ✓ Required

3.4 Admin Review Endpoints (Protected by JWT + ADMIN role)

   GET /api/registration/admin/submissions
   - Purpose: List all pending registrations
   - Query: { type?: STUDENT|TEACHER, status?: SUBMITTED|UNDER_REVIEW, page?: number, limit?: number }
   - Response: { submissions: [], total: number, page: number }
   - Auth: JWT + ADMIN role
   - Status: 200 | 401 | 403
   - HTTPS: ✓ Required

   GET /api/registration/admin/submission/:submissionId
   - Purpose: View detailed registration submission
   - Auth: JWT + ADMIN role
   - Response: { submission: {...}, documents: [...] }
   - Status: 200 | 404 | 403
   - HTTPS: ✓ Required

   POST /api/registration/admin/approve/:submissionId
   - Purpose: Approve registration and create user account
   - Auth: JWT + ADMIN role
   - Body: { notes?: string, tempPassword?: string }
   - Response: { success: boolean, userId: string, message: string }
   - Status: 200 | 400 | 403 | 404
   - HTTPS: ✓ Required

   POST /api/registration/admin/reject/:submissionId
   - Purpose: Reject registration with reason
   - Auth: JWT + ADMIN role
   - Body: { reason: string (required) }
   - Response: { success: boolean, message: string }
   - Status: 200 | 400 | 403 | 404
   - HTTPS: ✓ Required

3.5 File Upload Endpoints

   POST /api/registration/upload
   - Purpose: Upload registration documents (temporary)
   - Auth: Optional (temp token or anonymous)
   - Body: FormData { file: File, submissionId?: string }
   - Response: { fileName: string, fileUrl: string, size: number }
   - Status: 201 | 400 | 413 (too large)
   - HTTPS: ✓ Required
   - File Size Limits:
     * Individual: 5MB max
     * Total per submission: 20MB max
     * Allowed: PDF, JPEG, PNG, DOCX

   DELETE /api/registration/upload/:fileId
   - Purpose: Delete temporary uploaded file
   - Auth: Optional (temp token or owner)
   - Response: { success: boolean }
   - Status: 200 | 404
   - HTTPS: ✓ Required

5. SECURITY REQUIREMENTS
================================================================================

5.1 HTTPS/TLS
   - ALL traffic must be HTTPS
   - Self-signed certificates in development (backend/certs/)
   - Valid certificates in production
   - HSTS header: Strict-Transport-Security: max-age=31536000
   - SSL/TLS version: 1.2 minimum
   - Cipher suites: Modern, strong only

5.2 Authentication & Authorization
   - OTP via email (6 digits, 5-minute expiry)
   - JWT tokens for authenticated endpoints
   - Temp token for registration form (short-lived, ~30 minutes)
   - Session management: HttpOnly cookies for JWT (if used)
   - Role-based access: ADMIN, TEACHER, STUDENT
   - Rate limiting: 5 OTP requests per email/hour, 100 requests/minute per IP

5.3 Data Protection
   - Input validation on ALL endpoints (express-validator)
   - SQL injection prevention (Prisma parameterized queries)
   - XSS prevention (sanitize outputs, Content-Security-Policy headers)
   - CSRF tokens for form submissions
   - Password hashing: bcrypt (for temp passwords)
   - Data encryption at rest (if PII stored): application-level or DB-level
   - File upload security:
     * Whitelist file types: PDF, JPEG, PNG, DOCX
     * Scan files for malware (optional: ClamAV integration)
     * Store outside web root
     * Random file names (UUID)
     * No direct path exposure

5.4 Audit & Logging
   - Log all registration submissions (timestamp, IP, user agent)
   - Log all admin approvals/rejections (admin user, reason, timestamp)
   - Log all failed OTP attempts
   - Sensitive data NOT logged (passwords, OTPs, full ID numbers)
   - Log retention: 90 days minimum
   - Log format: Structured (JSON) with timestamp, level, message

5.5 Privacy & Compliance
   - Privacy Policy: Displayed before registration
   - Data retention: Clear policy (when data is deleted)
   - User consent: Explicit checkbox for data processing
   - Right to deletion: API endpoint for users to request data removal
   - GDPR-like compliance (if applicable)
